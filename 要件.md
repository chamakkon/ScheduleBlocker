# AI実装用要件定義書

**プロジェクト名:** Daily Visual Planner
**バージョン:** v0
**対象環境:** macOS デスクトップアプリ
**実装方式:** Electron + React + TypeScript + Zustand + Tailwind CSS
**保存方式:** ローカルJSON
**Python利用:** v0では使わない

---

## 1. プロダクト概要

本アプリは、ユーザーが朝にその日の候補タスクを並べ、縦タイムライン上にドラッグ&ドロップで配置し、1日分の予定を視覚的に作成・調整・保存・確定・画像出力するためのローカルデスクトップアプリである。

このアプリは一般的なタスク管理ツールではなく、**1日単位の視覚的スケジューラ**である。
優先度管理、複雑なプロジェクト管理、通知、クラウド同期は v0 では対象外とする。

同一画面内に以下の2モードを持つ。

* 左: CandidatePane
* 右: TimelinePane

### 4.2 Finalized Mode

* 右のタイムラインを中心に表示

* 「編集」操作で Planning Mode に戻れる

### 4.3 確定条件

## 5. タイムライン仕様

### 5.1 表示範囲

* 07:00〜24:00

### 5.2 時間単位

* 内部管理単位: 15分
* 目盛り表示: 30分ごと
* 最小ブロック長: 15分

### 5.3 unit定義

* 07:00 を `startUnit = 0`
* 15分ごとに +1
* 24:00 は最終端
* 総unit数は 68

## 6. ドラッグ&ドロップ仕様

### 6.1 配置基準

**ドラッグ中コンポーネントの上辺**が、ドロップ時にどの時間 index に一致しているかを開始時刻とする。

マウスカーソル位置そのものではなく、コンポーネント上辺を基準にする。

### 6.2 候補タスク → タイムライン

* 候補タスクを左ペインから右ペインへドラッグ
* ドロップ時に `startUnit` を決定
* `estimatedDurationMin` がある場合はその長さで block を作成
* ない場合はデフォルト 60分で block を作成

### 6.3 ブロック移動

* ブロック本体をドラッグ
* ドロップ時のコンポーネント上辺位置から `newStartUnit` を決定

### 6.4 ブロック上端リサイズ

* 上端ドラッグで開始時刻を変更
* 終了時刻は固定
* 15分単位で吸着

### 6.5 ブロック下端リサイズ

* 下端ドラッグで終了時刻を変更
* 開始時刻は固定
* 15分単位で吸着

### 6.6 スナップ

* すべて 15分単位で吸着する

---

## 7. 衝突ルール

### 7.1 基本方針

* 最終状態でブロック同士の重なりは許可しない
* ただし、衝突時はエラーにせず **自動シフト** する

### 7.2 自動シフトルール

配置または移動後に既存ブロックと重なる場合、

1. まず下方向に最も近い空きスロットを探す
2. 下方向に空きがなければ上方向を探す
3. どちらにも収まらなければ元位置に戻す

### 7.3 実装方針

* `findNearestAvailableSlot()` を実装する
* 対象ブロック以外の blocks をソートしてギャップ探索する

## 8. 候補タスク仕様

### 8.1 TaskCandidate

候補タスクは DailyPlan 単位で保持する。
グローバルライブラリとしては持たない。
固定候補テンプレートは設定からコピーして当日の候補へ流し込む。

### 8.2 フィールド

* title: 必須
* subTasks: 0〜3件
* colorId: 必須
* estimatedDurationMin: optional
* note: optional

### 8.3 priority は持たない

priority は v0 では不要。

---

## 9. 色仕様

### 9.1 基本思想

カテゴリは論理カテゴリではなく、まず **色** として扱う。

### 9.2 ColorSetting

* `id`
* `hex`
* `label?`

例:

* 青 = 研究
* 赤 = 運動
* 緑 = 雑務

ただし label は optional。

## 15. IPC API

renderer は `window.appApi` 経由でのみ Electron 機能にアクセスする。
`fs` や Electron API を renderer に直接書かないこと。

### 15.1 dailyPlan API

* `loadTodayPlan(): Promise<DailyPlan | null>`
* `createTodayPlanIfMissing(): Promise<DailyPlan>`
* `saveTodayPlan(plan: DailyPlan): Promise<void>`

### 15.2 settings API

* `loadSettings(): Promise<{ colors: ColorSetting[]; fixedTemplates: FixedCandidateTemplate[] }>`
* `saveSettings(payload): Promise<void>`

### 15.3 export API

* `exportTimelinePng(payload: { dataUrl: string; date: string }): Promise<{ savedPath: string }>`

## 17. PNG出力仕様

### 17.1 出力対象

* 右ペインの完成済みタイムラインのみ
* 左ペイン、ボタン、編集UIは含めない

### 17.2 表示ルール

* タイトルは必須表示
* サブタスクは横幅・高さに余裕がある場合のみ表示
* 余裕がなければ省略

### 17.3 実装方針

* renderer 側で対象 DOM を `html-to-image` 等で dataUrl に変換
* main 側で PNG として保存

## 22. 完了条件

v0 完了の条件は次の通り。

1. 当日の候補を作れる
2. 候補をタイムラインへドラッグ配置できる
3. ブロックをドラッグ移動できる
4. ブロックを上下リサイズできる
5. ブロック重複時に自動シフトする
6. 保存して再起動後も当日データが復元される
7. 確定後、その日の plan がトップ表示される
8. PNG出力できる
9. 設定画面で色と固定候補を管理できる

## 23. AIへの実装指示

以下を厳守すること。

* v0 では機能追加提案をしない
* Google Calendar や Slack を先回り実装しない
* Python は使わない
* 型を必ず shared に置く
* renderer に fs / Electron API を直接書かない
* ドラッグ仕様は「コンポーネント上辺基準」を厳守する
* 衝突時は「下方向優先、次に上方向」の自動シフトを実装する
* 候補は DailyPlan 単位で保持する
* priority は実装しない
