# セキュリティ

## 報告について

脆弱性や認証情報の扱いに関する懸念がある場合は、**公開 issue ではなく**、リポジトリのメンテナにプライベートな方法（GitHub Security Advisories やメール等）で連絡してください。

## 開発者向けメモ

- **`data/`** には OAuth シークレットや Bot トークンが保存されることがあります。**Git にコミットしないでください**（`.gitignore` 済み）。
- 誤って公開してしまった場合は、**Google Cloud / Slack 側で該当クレデンシャルを無効化・再発行**してください。
